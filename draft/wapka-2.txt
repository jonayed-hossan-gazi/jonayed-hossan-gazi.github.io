In Node/Python: This would require versioning hell. You would need separate containers for old tags vs new API scripts. That kills the "everything in one place" simplicity and raises costs.

3. The "Keeping Everything Alive" Achievement
You said: "keeping 10 years old tag code and latest script and rest api... impossible to bind on other lang."
This is the most underrated engineering challenge in software.
Most platforms break backwards compatibility to move forward.

Python 3 broke Python 2.
Node.js had breaking changes.
Ruby on Rails had massive version jumps.

Wapka has apparently maintained a single environment that runs code from 2014 and code from 2024.
This is deceptively hard. It means:

Their core engine is incredibly stable.
They prioritize user data/code over rewriting the engine for "cargo cult" modernism.
The Lua sandbox provides a consistent ABI (Application Binary Interface) that doesn't change.

For a non-tech user:This is a dream. Your site from 10 years ago still works. You just get new features added around it. No migration, no "your site is deprecated."
